What Is Docker?

  • Docker was written as a formalization and enhancement of current containerization practices

    •

  • Containers have been around for a LONG time. Examples include LPAR's, Solaris Zones, LXC, OpenVZ, etc.

    •

What Is Docker (continued)

  • Docker provides a framework, toolkits and repositories to supercede existing solutions
    •
  • Docker's philosophy revolves around the Shipping trade...containers go on barges, and barges transport containers, not boxes.
    •
  • With Docker, software is packaged into containers that transition dev, QA and production environments.
    •

Docker Deployments...how do they work?

  • Base image built or fetched from Docker Hub
    •
  • DockerFile created with self-documenting build-steps to download and assemble application and required pieces
    •
  • Docker container is built using docker daemon and uploaded to local repository or Docker Hub
    •
  • Target servers download container from repository and launched
    •

Deployment Breakdown

  • docker run [image] = Downloads Docker image from Docker Hub and Runs it
    •
  • docker pull [image] = Downloads new versions of docker images.
  • All docker images are pre-packaged --- everything you need is either built-in or passed in via -volume option
    •
  • Docker images accept external volume mounts for persistent data/config storage
    •

Docker Security

  • Docker relies on cgroups and limits to prevent processes from interacting with each other
    •
  • cgroups places a virtual fence/partition around each container, unless they are linked together
    •
  • linking docker containers allows for use of Docker Volumes (disk containers) but aren't common in the wild.
    •
  • Networking is also controlled via specific port mappings, ports are explicitly exposed and mapped to host
    •

    Docker Security (continued)

    • Docker containers run as root typically, but are confined to their specific 'chroot'
      •
    • Docker environments are typically very sparse, and only offer what is necessary to run intended applications.
      •
    • Since Docker incorporates a thin Linux Runtime and application dependencies, updates can occur to applications without taking hosts offline.
      •
